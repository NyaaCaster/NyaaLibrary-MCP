# Skill: sync-blueprint

## 描述

维护和同步项目开发任务蓝图。

## 触发

- 完成一个里程碑时
- 需要更新任务进度时
- 用户请求查看或更新蓝图时

## 蓝图文件

`.docs/BLUEPRINT.md`

## 状态符号

- ⬜ 未开始
- 🟡 进行中
- ✅ 已完成

## 里程碑完成流程

1. **验证**：确认所有 `- [ ]` 子项实际完成
2. **更新状态**：
   - 将里程碑符号改为 ✅
   - 添加 `_完成于 YYYY-MM-DD_`
   - 所有子项改为 `- [x]`
3. **变更日志**：在变更日志表格追加记录
4. **推进**：将下一个里程碑从 ⬜ 改为 🟡
5. **代码签名验证**：确认 `BLESSING` 嵌入点完整
   - `src/version.ts` 导出存在
   - `index.html` 的 `data-blessing` 属性存在
   - `src/main.tsx` 的 `console.log(BLESSING)` 存在
6. **提交**：通过 `commit-push` skill 提交变更

## 禁止操作

- 勾选未完成的子项
- 单次提交跨越多个里程碑
- 遗漏变更日志条目
